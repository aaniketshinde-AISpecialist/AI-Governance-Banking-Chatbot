# AI Governance Assessment — Banking Customer Support Chatbot

## Project Summary

This project demonstrates an end-to-end AI Governance assessment conducted for a retail banking customer support chatbot powered by a third-party Large Language Model (LLM).

The assessment focused on:
- AI risk identification
- Governance and control design
- Human oversight mechanisms
- Vendor governance
- Residual risk analysis
- Monitoring and KPI design
- Alignment with the NIST AI Risk Management Framework

---

## Key Governance Areas Covered

### AI Risk Assessment
- Hallucination risk
- Third-party data leakage
- Unauthorized autonomous actions
- Data retention violations
- Escalation governance failures
- Operational resilience risks

### Governance Controls
- PII redaction mechanisms
- API-grounded response architecture
- Automated retention enforcement
- Human-in-the-loop escalation workflows
- Outbound payload inspection
- Governance monitoring metrics

## Project Outcome

The assessment concluded that AI-enabled banking support systems can operate within an acceptable risk posture when supported by:
- layered governance controls,
- operational monitoring,
- human oversight,
- and continuous governance assurance.
